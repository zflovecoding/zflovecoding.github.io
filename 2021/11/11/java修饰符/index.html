java修饰符

2021-11-11
java
字数统计: 1k   |   阅读时长≈ 3 分钟

java中有多种修饰符，主要分为访问修饰符和非访问修饰符，修饰符的主要作用是定义类，方法和变量。

访问修饰符

访问修饰符是最常用的，每天写java语句只要定义变量，创建实例就会涉及到。主要有以下四个访问修饰符。

  • public：公开的，对所有类可见。适用对象：类，变量，接口，方法
  • private：私有的，只有同一类内可见。适用对象：变量，方法。不可以修饰类（外部类）
  • default：就是我们正常不写修饰符的默认选项。在同一包内可见。适用：类，变量，接口，方法
  • protected：受保护的，同一包内的类和所有子类可访问。适用：变量，方法。不可以修饰类（外部类）

注意：

protected可见性在于两点：

  • 基类的 protected 成员是包内可见的，并且对子类可见；
  • 若子类与基类不在同一包中，那么在子类中，子类实例可以访问其从基类继承而来的protected方法，而不能访问基类实例的protected方法。

修饰符继承规则：

  • 父类中public方法，子类必须为public
  • 父类中protected方法，子类protected或者public
  • 父类中private方法，不可继承

非访问修饰符

  • static: 修饰类方法与类变量。
  • final：修饰类，方法，变量。修饰的类不可被继承；方法不可被继承类重新定义；变量为常量，不可更改。
  • abstract: 修饰抽象类和抽象方法。
  • synchronized：声明的方法同一时间只能被一个线程访问。
  • volatile：修饰的成员变量每次被线程访问，都强制从内存中重新读取该成员变量的值。
  • transient：序列化的对象中被 transient 修饰的实例变量会被JVM跳过该特定的变量。

static：

类变量：和类一起加载，一起毁灭。类变量（静态变量）只有一份拷贝，无论该类实例化多少对象，是共享的。局部变量不可以被声明为static。

类方法：独立于对象，也叫静态方法，不能使用类的非静态变量，从参数列表得到数据再计算。

使用类名.变量名/类名.方法名访问。

final:

被 final 修饰的实例变量必须显式指定初始值。通常和 static 修饰符一起使用来创建类常量。

父类中的 final 方法可以被子类继承，但是不能被子类重写。

声明 final 方法的主要目的是防止该方法的内容被修改。

abstract:

抽象类不能用来实例化对象，声明抽象类的唯一目的是为了将来对该类进行扩充。

包含抽象方法的类必须是抽象类。

抽象类可以包含抽象方法和非抽象方法。

synchronized:

synchronized 修饰符可以应用于四个访问修饰符。常用于控制线程安全，之后会单写一篇博客介绍synchronized。

public synchronized void show(){}

volatile:

volatile 修饰的成员变量在每次被线程访问时，都强制从共享内存中重新读取该成员变量的值。而且，当成员变量发生变化时，会强制线程将变化值回写到共享内存。这样在任何时刻，两个不同的线程总是看到某个成员变量的同一个值。

一个 volatile 对象引用可能是 null。
